# Kairat Security — лендинг

Одностраничный сайт охранного агентства «Kairat Security» (Алматы, работает с 2016 года).
Язык сайта и общения — русский. Отвечай кратко и конкретно.

## Текущее состояние

- `index.html` — готовы секции: хиро, услуги, о компании, география, оставьте заявку, наши клиенты, финальный CTA. Хиро — эталон стиля кода: новые секции верстай в той же структуре и конвенциях.
- `fonts/` — файлы шрифтов Druk Cyr и SF Pro Display, @font-face уже настроен.
- `assets/` — сюда пользователь экспортирует картинки из Figma вручную. Сейчас добавлены `geography-map.png`, `clients-bg.png`, `client-logo-*.png`, `client-letter-*.png`.
- Дизайн: новый рабочий файл https://www.figma.com/design/MOlmu34600PdAL3ej45JhW/Untitled (Figma MCP авторизован через `241491@astanait.edu.kz`). Готовые секции: хиро, услуги, о компании, география, оставьте заявку (`1:1176`), наши клиенты (`1:1115`), финальный CTA (`1:1069`, с нижней надписью «Охранное агентство»); остальные секции пользователь даёт ссылками по одной.

## Рабочий процесс (важно, выработан опытом)

1. **По одной секции за раз.** Пользователь даёт ссылку на конкретный фрейм. Не бери весь лендинг целиком.
2. **Figma MCP + скриншот вместе.** Вытяни design context по фрейму И запроси/используй скриншот фрейма как визуальный эталон. После вёрстки сравни результат со скриншотом и исправь расхождения сам, до показа пользователю.
3. **Точные значения из Figma.** Размеры шрифтов, отступы, цвета в hex, border-radius — бери как есть. Не округляй, не заменяй на «примерно похожие», не подставляй свои привычные значения.
4. **Ассеты не тяни из MCP в финальный код.** URL вида figma.com/api/mcp/asset/... живут 7 дней. Проси пользователя экспортировать картинки в `assets/` (иконки — SVG, фото — WebP/PNG 2x) и ссылайся на локальные файлы. Простые иконки (стрелки, шевроны, плюсы) рисуй inline-SVG сам.
5. **Итерация по скриншотам.** Если пользователь присылает скриншот из браузера с замечаниями — правь по нему точечно, не перелопачивая остальное.

## Конвенции кода

- Один файл `index.html`: CSS в `<style>`, ванильный JS в `<script>` внизу. Без фреймворков и сборки.
- Классы в BEM-стиле: `.hero__title`, `.topbar__inner`.
- CSS-переменные в `:root`: `--font-display` (Druk Cyr → Oswald), `--font-body` (SF Pro Display → Inter), `--white`, `--hairline` (rgba(255,255,255,0.45) — тонкие линии из макета), `--overlay`, `--pad-side` (60px десктоп / 20px мобилка). Новые общие значения — тоже в переменные.
- Текст в HTML — с обычным регистром, капс через `text-transform: uppercase` (как в хиро).
- Тонкие линии макета (0.3px в Figma) — `1px solid var(--hairline)`.

## Адаптив

Мобильного макета НЕТ — мобильные решения принимаются в коде по паттернам хиро:
- Брейкпоинт 768px; `--pad-side` меняется сам.
- Колонки складываются в одну, сетки → 1 колонка.
- Заголовки через `clamp()` (в хиро: 60px → clamp(38px, 10.5vw, 56px)).
- Основной текст не меньше 16px, тап-таргеты не меньше 44px.
- Бургер-меню уже реализовано в хиро — не дублировать.
- После каждой секции проверяй вид на ~390px.

## Бренд и тексты

- В маркетинге — «Kairat Security» латиницей; формулировка «Охранное агентство «Kairat Security»». «ТОО «Kairat Security»» — только в юридических документах/реквизитах.
- Частая опечатка в макете: «агенство» → правильно «агентство». Исправляй в коде, если встретишь.
- Тексты бери из макета, но проверяй орфографию.
