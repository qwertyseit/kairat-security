# Kairat Security — лендинг (инструкции для агента)

Одностраничный сайт охранного агентства «Kairat Security» (Алматы, с 2016 года).
Язык сайта и общения — русский. Отвечай кратко и конкретно.

> Полная версия правил — в `CLAUDE.md` (этот файл — та же суть плюс актуальный
> статус работы). Если правки в двух файлах расходятся — правь оба.

## Стек и структура

- **Без фреймворков и сборки.** Один файл `index.html`: весь CSS в `<style>` в
  `<head>`, ванильный JS в `<script>` внизу.
- `fonts/` — Druk Cyr (Medium/Bold), SF Pro Display (Regular/Medium). `@font-face`
  настроены. Bebas Neue Pro (крупные цифры) файла пока нет — грузится фолбэк.
- `assets/` — картинки (пользователь экспортирует из Figma вручную).
- `styles.css` в корне — **устаревший** (остаток раннего билда, не подключён).
  Не редактируй его; вся вёрстка живёт внутри `index.html`.

## Рабочий процесс (важно)

1. **По одной секции за раз.** Пользователь даёт ссылку на конкретный фрейм Figma.
   Не берись за весь лендинг сразу.
2. **Figma MCP + скриншот вместе.** Вытяни design context по фрейму и сверься со
   скриншотом. После вёрстки сам сравни результат с макетом и поправь расхождения
   до показа пользователю.
3. **Точные значения из Figma** — размеры, отступы, цвета (hex), радиусы бери как
   есть, не округляй и не заменяй «на глаз».
4. **Ассеты не тяни из MCP в финальный код** — ссылки `figma.com/api/mcp/asset/...`
   живут 7 дней. Проси экспортировать картинки в `assets/` (иконки — SVG, фото —
   WebP/PNG 2x), ссылайся на локальные файлы. Места временных ссылок помечены
   комментарием `<!-- ЗАМЕНИ ... -->`. Простые иконки (стрелки, шевроны, плюсы)
   рисуй inline-SVG сам.
5. **Итерация по скриншотам** — если пользователь прислал скрин из браузера с
   замечаниями, правь точечно, не перелопачивая остальное.

## Конвенции кода

- Классы в BEM-стиле: `.hero__title`, `.services__row`, `.about__stats`.
- CSS-переменные в `:root`:
  - `--font-display` — Druk Cyr → Oswald (заголовки).
  - `--font-body` — SF Pro Display → Inter (текст).
  - `--font-mono` — PT Mono (лейблы секций `[ ... ]`).
  - `--font-numbers` — Bebas Neue Pro → Bebas Neue (крупные цифры-показатели).
  - `--white`, `--hairline` (rgba(255,255,255,0.45)), `--line` (0.3 — рамки),
    `--line-dim` (0.1 — разделители строк), `--label` (#585858 — цвет лейблов),
    `--overlay` (rgba(0,0,0,0.4)), `--pad-side` (60px десктоп / 20px мобилка).
  - Новые общие значения — тоже выноси в переменные.
- Общий класс `.section-label` — лейбл `[ Название ]` (PT Mono, 12px, `--label`).
  Используй его во всех секциях.
- Текст в HTML пиши обычным регистром, капс делай через `text-transform: uppercase`.
- Неразрывные пробелы (`&nbsp;`) после предлогов/союзов в заголовках и абзацах.
- Тонкие линии макета (0.3px в Figma) → `1px solid var(--line)`.
- Тексты бери из макета, но проверяй орфографию.

## Бренд и тексты

- В маркетинге — «Kairat Security» латиницей; «Охранное агентство «Kairat Security»».
  «ТОО «Kairat Security»» — только в юр-документах/реквизитах.
- Частая опечатка макета: «агенство» → «агентство». Исправляй.
- Прямые кавычки `"..."` → ёлочки «...».

## Адаптив

Мобильного макета нет — решения по паттернам хиро:
- Брейкпоинт 768px; `--pad-side` меняется сам.
- Колонки складываются в одну, сетки → 1 колонка.
- Заголовки через `clamp()` (в хиро: 60px → `clamp(38px, 10.5vw, 56px)`).
- Основной текст не меньше 16px, тап-таргеты не меньше 44px.
- Бургер-меню уже реализовано в хиро — не дублировать.
- После каждой секции проверяй вид на ~390px (нет горизонтального скролла).

## Навигация и якоря

Меню: Главная `#top`, Услуги `#services`, Наши клиенты `#clients`, Контакты
`#contacts`. Новым секциям ставь `id`. CTA «Оставить заявку» ведёт на `#contacts`
(форма/WhatsApp — за пользователем, уточни при вёрстке контактов).

## Предпросмотр

Просто открой `index.html` в браузере, либо подними статику из корня проекта:
`npx http-server -p 8123 -c-1 .` → http://localhost:8123
(`.claude/launch.json` — это конфиг предпросмотра Claude Code, Codex его не использует.)

### Как проверять результат
Скриншот-инструменты предпросмотра могут быть недоступны — тогда сверяйся с макетом
численно: через геометрию элементов (`getBoundingClientRect`), computed styles и
измерение ширины текста, чтобы подтвердить шрифт/размеры/выравнивание и отсутствие
горизонтального скролла на 1200px и ~390px.
